# HANDOFF GUIDE — 開発チーム編成・運営の手引き

このドキュメントは、**プロダクトオーナー（非エンジニア）が開発チームを編成して Claude Code で開発を進めるための実践ガイド**です。

---

## このガイドの位置づけ

あなた（祐介さん）はプロダクトオーナーで、自分でコードを書く予定はない。代わりに：
- 開発者を1〜数名雇う
- 雇った開発者は **Claude Code** を使って効率的に開発
- あなたは**仕様策定・予算管理・ユーザー対応**に集中

このガイドはそのための手順書です。

## 3. Claude Code を使う開発者の特徴

Claude Code を使うエンジニアは：
- 1人で2〜3人分の生産性を出す（コード生成・テスト・ドキュメント化が高速）
- 細かい仕様は CLAUDE.md に書いておけば自動的に理解
- レビューと意思決定が主な人間の役割

**確認すべきこと：**
- Claude Code の利用歴（最低3ヶ月以上推奨）
- どんなプロジェクトで使ったか
- どう活用しているか（聞くと熱量がわかる）

---

## 4. 契約のポイント

### 必須条項
- **成果物の権利**：完成コードはオーナー（あなた）に帰属
- **NDA**：β版コード・仕様・コーチデータの守秘
- **検収条件**：ROADMAP.md の Phase 1 DoD（完了の定義）を満たすこと
- **支払い**：マイルストーンごとに分割（着手金30%、中間30%、完了40% など）

### 推奨条項
- 月次レポート（進捗・課題）
- 週次 Zoom MTG（30分）
- 緊急時対応の連絡先
- Phase 2 への継続オプション

---

## 5. オーナーとしての日常業務

### 毎週やること（合計2〜3時間）
- [ ] 週次 MTG（30分）：進捗確認、優先順位調整
- [ ] PR レビュー（1〜2時間）：開発者のPRを軽くチェック
  - ※ コードの中身は読めなくていい
  - ※ Claude Code がレビューサマリーを出してくれる
  - ※ 「これで何ができるようになるか」を確認
- [ ] テスト（30分）：開発中のステージング URL で実際に触ってフィードバック

### 毎月やること（合計1日）
- [ ] β版コーチ20人にアンケート（Google Forms）
- [ ] フィードバック整理→次月の優先順位策定
- [ ] 月次レポート受領＆支払い処理
- [ ] 次月予算の確定

### 自分でやらないこと
- ❌ コードを書く
- ❌ Firebase / Vercel 設定（開発者に任せる）
- ❌ サーバートラブル対応（緊急連絡網で開発者に丸投げ）

## 6. Claude Code をオーナー自身が使うケース

開発者を雇わず、自分で進める場合のヒント：

### できること
- ✅ ドキュメント整備（このプロジェクトはまさにそう）
- ✅ 簡単なバグ修正
- ✅ コーチへの説明資料作成
- ✅ マーケ素材（LP、ブログ、SNS投稿）

### 難しいこと
- ⚠️ アーキテクチャ判断（Firebase の設定など）
- ⚠️ セキュリティ実装
- ⚠️ 決済システム
- ⚠️ 本番障害対応

→ **混合パターン**もアリ：オーナーがドキュメント・マーケ・コーチ対応、開発者がコード。

---

## 7. 開発者が辞めたら（リスク管理）

最大のリスクは「開発者が突然消える」です。対策：

1. **CLAUDE.md を最新に保つ**：これがあれば次の開発者が同じスピードで参入可能
2. **コードの読みやすさ重視**：契約書に「ドキュメント整備」を含める
3. **複数人に依頼検討**：1人だけでなく2人体制で進める
4. **段階的引き継ぎ**：契約終了の1ヶ月前から次の人を準備

## 9. 経理・税務

### 個人で運営する場合
- **個人事業主開業届**を提出（屋号「Cheer Choreo」など）
- 業務委託費は「外注費」として経費計上
- 売上が¥1,000万超で消費税課税事業者

### 法人化のタイミング
- 売上 ¥800万〜¥1,000万超えたら検討
- 海外展開を本気で考えるなら早めに法人化

### 必要なツール
- 会計：freee / マネーフォワード（月¥3,000〜）
- 請求書発行：マネーフォワード or Misoca
- 銀行口座：屋号付き口座 or 法人口座
- 弁護士：利用規約レビューで1回スポット契約

## 12. 最後に：成功するオーナーの条件

このプロダクトを成功させる鍵は：

1. **コーチへの理解**：自分でβ版を使ってもらい、生の声を聞く
2. **意思決定のスピード**：完璧主義より「決めて進める」
3. **任せる勇気**：開発の細かいことは開発者に任せる
4. **継続力**：3〜6ヶ月は売上ゼロでも続ける覚悟
5. **コミュニティ作り**：β版コーチたちを「初期メンバー」として大切にする

頑張ってください！困ったら CLAUDE.md と ROADMAP.md を読み返せば、方向性はずれません。
