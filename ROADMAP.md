# ROADMAP — 開発フェーズ計画

このドキュメントは Cheer Choreo の開発を「Phase 0 ＝ 現状」から商用化までの段階に分けて整理したものです。

---

## 全体観

| Phase | 目標 | 期間目安 | 開発工数 | 状態 |
|---|---|---|---|---|
| **Phase 0** | β版（単一HTML）を完成 | 完了 | — | ✅ 完了 |
| **Phase 1** | MVP（Web SaaS・個人向け） | 2〜3ヶ月 | 1.5〜3人月 | ⏳ 次着手 |
| **Phase 2** | マルチチーム・コーチ協業 | 2ヶ月 | 1〜2人月 | 🔜 |
| **Phase 3** | 課金・サブスク | 1〜2ヶ月 | 1人月 | 🔜 |
| **Phase 4** | 拡張機能・モバイルアプリ | 継続 | 継続 | 🔜 |

「人月」は1人のフルタイム開発者が1ヶ月作業した場合の目安です。Claude Code 活用で 1.5〜2倍速くなる想定。

## ⏳ Phase 1: MVP（Web SaaS・個人向け）

**目標：個人コーチが自分のアカウントを作って、自分の振付を保存・編集・共有できる**

### スコープ
1. **プロジェクト基盤**
   - Next.js 15 + TypeScript + Tailwind
   - Firebase（Auth, Firestore, Storage）
   - Vercel デプロイ
   - GitHub Actions CI/CD

2. **認証**
   - Google ログイン
   - Apple ログイン
   - メール+パスワード
   - ログアウト

3. **オンボーディング**
   - 初回ユーザー：チーム名入力 → 最初の振付テンプレート選択

4. **β版機能の再実装（コア）**
   - フロア（USA / JCA, 真上 / 正面）
   - メンバー配置・ドラッグ・タップ移動
   - スタンツ作成・編集
   - 隊形遷移・再生
   - 音源アップロード・BPM検出・速度・ループ
   - 採点判定（簡易ルール）
   - Undo/Redo
   - ヘルプモーダル
   - 操作カード
   - フォーメーションプリセット

5. **クラウド保存**
   - 振付一覧画面
   - 振付詳細・編集画面
   - 自動保存（編集ごとに Firestore に書き込み）
   - メンバー管理画面

6. **共有（簡易版）**
   - 「閲覧専用URL」を発行
   - QRコード自動生成
   - SNSシェアボタン

7. **基本的なUI/UX**
   - レスポンシブ（モバイル・iPad・PC）
   - ダークモード
   - 日本語ローカライズ完璧
   - エラーハンドリング・ローディング表示

### スコープ外（次フェーズ）
- チームに複数コーチ
- リアルタイム共同編集
- 有料プラン
- 動画書き出し

### 完了の定義（DoD）
- [ ] サインアップから振付作成→共有まで一連の流れが動く
- [ ] iPhone Safari / iPad Safari / Chrome（PC/Mobile）で動作確認済
- [ ] β版コーチ20人に使ってもらいフィードバック収集
- [ ] エラー率 < 1%、ページロード < 3秒

### 推奨人員・工数
- フルスタックエンジニア 1名（Next.js + Firebase 経験者）
- 期間：2〜3ヶ月（Claude Code 活用前提）
- 工数：1.5〜3人月

### 採用先候補
- Lancers / CrowdWorks（個人）：月額40〜80万円
- Upwork（海外個人）：月額20〜60万円
- 日本のスタートアップ系受託：月額80〜150万円

## 🔜 Phase 2: マルチチーム・コーチ協業

**目標：複数のコーチが同じチームで振付を共同編集できる**

### スコープ
1. **チーム機能拡張**
   - チームに複数コーチを招待（招待リンク方式）
   - コーチの権限管理（オーナー、編集者、閲覧者）
   - 1ユーザーが複数チームに所属可能

2. **リアルタイム共同編集**
   - Firestore `onSnapshot` でフレーム変更を即時反映
   - 編集中の他コーチのカーソル表示（オプション）
   - 楽観的更新

3. **共有・通知機能**
   - 振付の編集権限を細かく付与
   - チームメンバー間でのコメント機能
   - メール通知（編集された、コメントされた）

4. **チーム管理画面**
   - メンバー一覧
   - 役割編集
   - 招待管理
   - チーム設定

### 完了の定義
- [ ] 3〜5人のコーチが同時に同じ振付を編集してもデータ破損なし
- [ ] 招待→受諾フローが10秒以内
- [ ] AllStarジムなど実チームでパイロット運用成功

### 推奨人員・工数
- フルスタックエンジニア 1〜2名
- 期間：2ヶ月
- 工数：1〜2人月

## 🔜 Phase 3: 課金・サブスク

**目標：Free・Pro・Team プランで収益化**

### スコープ
1. **Stripe 統合**
   - Stripe Customer / Subscription 作成
   - Firebase Extensions の Stripe Payments を利用
   - 日本円・米ドル両対応

2. **プラン制限**
   - Free：1チーム / 5メンバー / 3振付 / 30秒音源
   - Pro：無制限 + 動画書き出し（¥980/月、¥9,800/年）
   - Team：複数コーチ + 共同編集（¥4,800/月、¥48,000/年）

3. **アップグレードUI**
   - 制限到達時の案内モーダル
   - プラン比較ページ
   - チェックアウトフロー

4. **請求・領収書**
   - 自動領収書発行（Stripe）
   - インボイス制度対応（日本）
   - 解約・プラン変更フロー

5. **アナリティクス**
   - LTV / Churn / MRR 追跡
   - Posthog でファネル分析

### 完了の定義
- [ ] サブスク開始→自動継続→解約まで正常動作
- [ ] 初期100人の有料ユーザー獲得
- [ ] MRR ¥100,000 突破

### 推奨人員・工数
- フルスタックエンジニア 1名 + マーケ
- 期間：1〜2ヶ月
- 工数：1人月

## 🔜 Phase 4: 拡張機能・モバイルアプリ

**目標：競合との差別化・成熟プロダクト化**

### 候補機能（優先順位は別途決定）

#### A. メディア機能
- 動画書き出し（MP4）：サーバー側で ffmpeg 処理
- PDF書き出し：隊形図の印刷用
- スクリーンレコーディング録画

#### B. 採点ロジック強化
- JCA / USA / NCA / NDA の公式規定に対応
- 大会別テンプレート（高校選手権、大学選手権など）
- 公認審判監修

#### C. AI 補助
- 「この曲に合う隊形を3案」生成（Claude API活用）
- 構成条件を満たす配置の自動提案
- 演技動画から振付の自動採譜

#### D. モバイルネイティブアプリ
- React Native（コードベース共有）
- iOS / Android App Store 配信
- オフライン編集
- AirPlay / Chromecast 対応

#### E. コラボレーション強化
- 動画コメント（タイムスタンプ付き）
- 振付の Git 的バージョン管理
- AB比較（バージョンA vs B）

#### F. 大会・トーナメント対応
- 大会用テンプレート
- 採点項目の自動計算
- リハーサルモード（時間制限あり）

## 🚨 リスク・対策

| リスク | 対策 |
|---|---|
| JCA/USA の規定名を勝手に使用 | 「対応」「準拠」など曖昧表現、ロゴは絶対使わない |
| プライバシー漏洩（選手の名前・写真） | チーム内アクセス制御、保存先暗号化 |
| 開発者の引き抜き・退職 | ドキュメント整備（このCLAUDE.mdが基盤）、PRレビュー必須 |
| 競合参入 | 国内市場でのブランド先行・コーチ評価獲得を優先 |
| Firebase ベンダーロックイン | 将来の Supabase 移行を視野に、データアクセスは抽象化レイヤ越し |
